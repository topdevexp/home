# 代码走查

## 什么样的代码走查是我们所追求的

* 所有DEV都积极参加，QA可以选择参加
* 30分钟内结束
* 回顾完今天所有已提交的代码
* 提出的反馈大部分集中在整洁架构和代码重用上
* 代码lint规则没有随意改变，如需改变，要团队达成一致
* 不是不关心代码命名及规范，而是这些细节都要交给自动化工具，如[PMD][pmd], [LINT][lint]
* 昨天的反馈都已经在修改或已提交



[pmd]: https://en.wikipedia.org/wiki/PMD_(software)
[lint]: https://en.wikipedia.org/wiki/Lint_(software)
